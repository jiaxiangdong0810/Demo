# 音立方 云网通 — 蓝牙通信协议使用说明

## 一、设备概述

本文档描述了**音立方**蓝牙录音设备与 APP 之间的 BLE 通信协议。设备支持蓝牙（BLE）和 WiFi 双通道数据传输，具备录音、文件管理、OTA 升级等功能。

---

## 二、蓝牙服务与特征值 UUID

| 功能 | UUID |
|------|------|
| **蓝牙主服务** | `001120a0-2233-4455-6677-88995a5b5c5d` |
| **音频 Notify** | `001120a1-2233-4455-6677-88995a5b5c5d` |
| **指令 Notify** | `001120a3-2233-4455-6677-88995a5b5c5d` |
| **写入（Write）** | `001120a2-2233-4455-6677-88995a5b5c5d` |

> **说明：** APP 通过 Write 特征值发送指令给设备，设备通过 Notify 特征值返回响应和数据。

---

## 三、指令格式说明

- **APP → 设备指令前缀：** `YWT_BLE&...`
- **设备 → APP 响应前缀：** `YWT_DEV&...`
- 指令中的参数以 `&` 分隔，字符串参数通常用双引号包裹。

### 4.6 WiFi 控制

| APP 发送指令 | 功能说明 | 设备回复指令 | 回复说明 |
|---|---|---|---|
| `"YWT_BLE&WIFIO"` | 打开设备 WiFi | `"YWT_DEV&WIFIO"` | 设备 WiFi 已开启 |
| `"YWT_BLE&WIFIC"` | 关闭设备 WiFi | `"YWT_DEV&WIFIC"` | 设备 WiFi 已关闭 |
| `"YWT_BLE&WIFI"` | 获取 WiFi 名称和密码 | `"YWT_DEV&WIFI&SSID&PWD"` | SSID：WiFi 名称；PWD：WiFi 密码 |

**WiFi 状态码（STA）：**

| 状态码 | 含义 |
|--------|------|
| `0` | WiFi 关闭 |
| `1` | WiFi 已连接 |
| `2` | WiFi 未连接 |
| `3` | 等待 WiFi 开启 |
| `4` | 修改密码中 |
| `5` | OTA 升级中 |
| `6` | 密码修改成功，等待系统复位关机 |
| `7` | 自动关闭 |

## 五、特殊功能说明

### 5.1 BLE 实时音频流

当 APP 连接设备的 BLE 并监听**音频 Notify 端口**（`001120a1-...`）时，设备在录音状态下会实时向 APP 传输音频数据流，格式为 **MP3**。

### 5.2 BLE 同步录音文件

APP 端接收到 BLE 通道的数据包，**每帧数据均为有效数据**，需全部保存。

### 5.3 WiFi 同步录音文件

APP 端接收到 WiFi 通道的数据包，每帧数据均为有效数据，需全部保存。**最后5个字节** `0xBA 0x5A 0x02 0x8F 0x04` 为传输完成标志，APP 可据此判断传输结束。

### 5.4 MCU OTA 升级流程

1. APP 下发 `"YWT_BLE&OTA&LEN"` 指令
2. 设备回复 `"YWT_DEV&OTA"` 表示就绪
3. APP 发送 OTA 数据，**每帧 244 字节，帧间隔 ≥ 5ms**
4. 发送完成后下发 `"YWT_BLE&OT&OVER"`
5. ⚠️ **OTA 过程中禁止发送其他指令，否则会 OTA 失败**

### 5.5 BLE OTA WiFi 升级流程

1. APP 下发 `"YWT_BLE&WIFI&OTA&LEN"` 指令
2. 设备回复 `"YWT_DEV&OTA"` 表示就绪
3. APP 发送 WiFi 固件数据，**每帧 244 字节，帧间隔 ≥ 8ms（iOS 建议 20ms）**
4. 发送完成后下发 `"YWT_BLE&OT&OVER"`
5. 设备成功接收回复 `"YWT_DEV&OT&OVER"`，失败回复 `"YWT_DEV&OT&ERR"`
6. 设备进入 WiFi 升级阶段，状态变化：`"YWT_DEV&WIFIS&5"` → `"YWT_DEV&WIFIS&0"` 表示成功
7. WiFi 升级失败回复 `"YWT_DEV&OW&ERR"`
8. ⚠️ **OTA 过程中禁止发送其他指令，否则会 OTA 失败**

### 5.6 WiFi 使用注意事项

- **每次开启 WiFi 均为 AP 模式**
- 设备首次连接 APP 或重置后连接 APP，接收到密钥后会自动打开 WiFi 并设置密码：
  - WiFi 状态变为 `4`（修改密码中）
  - 设置成功后变为 `6`（等待复位关机）
  - 5 秒后自动关闭，整个过程约 **8 秒**
- WiFi 出问题时设备会自动复位，约需 **6 秒**
- WiFi 开启后 **30 秒** 内无设备连接将自动关闭
- WiFi 连接后断开，**5 秒** 内关闭
- BLE 断开连接时，WiFi 会自动关闭
- WiFi 状态为 `4`、`5`、`6` 时，无法通过指令关闭 WiFi

## 七、快速参考

### 常用操作流程

**开始录音：**
```
APP → "YWT_BLE&STA"
设备 → "YWT_DEV&STA&FNAME"  （返回录音文件名）
```

**停止录音：**
```
APP → "YWT_BLE&STO"
设备 → "YWT_DEV&STO"
```

**获取电量：**
```
APP → "YWT_BLE&BAT"
设备 → "YWT_DEV&BAT&RATE"
```

**获取文件列表：**
```
APP → "YWT_BLE&LIST_DIRS"
设备 → "YWT_DEV&DIRS&2025-08-13"  （依次返回每个文件夹）
设备 → "YWT_DEV&DIRS_SUM&5"        （文件夹总数）

APP → "YWT_BLE&LIST&2025-08-13"
设备 → "YWT_DEV&F&2025-08-13&录音1.mp3&120"  （依次返回每个文件及秒数）
设备 → "YWT_DEV&LIST&3"                        （文件总数）
```

**BLE 同步文件：**
```
APP → "YWT_BLE&U&2025-08-13&录音1.mp3"
设备 → "YWT_DEV&U&102400"   （文件大小 102400 字节）
设备 → 传输文件数据...
设备 → "YWT_DEV&OFF"          （传输完成）
```

**WiFi 同步文件：**
```
APP → "YWT_BLE&W&2025-08-13&录音1.mp3"
设备 → "YWT_DEV&W&102400"   （文件大小 102400 字节）
设备 → 传输文件数据...（最后 5 字节为 0xBA 0x5A 0x02 0x8F 0x04 表示完成）
设备 → "YWT_DEV&OFF"          （传输完成）
```
