# 音立方录音设备 BLE 实时传输 Demo 需求文档

## 一、项目背景

本项目用于面试 Demo 展示。

公司提供一个外设录音设备，该设备可以持续录音，并支持通过 BLE 与手机 App 通信。当前项目需要基于《音立方云网通_蓝牙通信协议_使用说明.md》实现一个 Android Demo，验证手机端能够连接设备，并实时接收设备传输过来的录音数据。

本 Demo 的重点不是完整商业 App，而是验证核心链路：

**手机扫描设备 → 蓝牙连接 → 订阅数据通道 → 接收实时音频数据 → 在手机端展示接收结果。**

## 二、项目目标

实现一个 Android 小 Demo，主要完成以下目标：

1. 能扫描并发现音立方录音设备。
2. 能通过 BLE 成功连接设备。
3. 能订阅设备的音频 Notify 通道和指令 Notify 通道。
4. 能通过 Write 特征值向设备发送基础控制指令。
5. 能实时接收设备推送的 MP3 音频流数据。
6. 能在手机界面展示连接状态、设备状态和实时数据接收情况。
7. 能展示基础录音控制能力，例如开始录音、停止录音、查询电量、查询存储。

## 三、协议依据

本项目依据文件：

`音立方云网通_蓝牙通信协议_使用说明.md`

核心 BLE UUID 如下：

| 功能 | UUID |
| --- | --- |
| 蓝牙主服务 | `001120a0-2233-4455-6677-88995a5b5c5d` |
| 音频 Notify | `001120a1-2233-4455-6677-88995a5b5c5d` |
| 写入 Write | `001120a2-2233-4455-6677-88995a5b5c5d` |
| 指令 Notify | `001120a3-2233-4455-6677-88995a5b5c5d` |

通信规则：

| 方向 | 格式 |
| --- | --- |
| App 发送给设备 | `YWT_BLE&...` |
| 设备回复给 App | `YWT_DEV&...` |

说明：

- App 通过 Write 特征值发送指令。
- 设备通过指令 Notify 返回指令响应。
- 设备通过音频 Notify 推送实时 MP3 音频数据。
- 指令数据和音频数据需要分开处理，不能把音频二进制数据当作文本指令解析。

## 四、核心功能需求

### 4.1 蓝牙权限与环境检查

App 启动后需要检查：

1. 手机是否支持蓝牙。
2. 蓝牙是否已开启。
3. 是否具备 BLE 扫描和连接权限。
4. Android 12 及以上需要处理 `BLUETOOTH_SCAN`、`BLUETOOTH_CONNECT` 权限。
5. Android 12 以下需要处理定位权限。

如果权限缺失，需要引导用户授权。

### 4.2 扫描设备

用户点击“扫描设备”后，App 开始扫描 BLE 设备。

扫描要求：

1. 优先按音立方主服务 UUID 过滤设备。
2. 扫描到设备后，在界面展示设备名称和 MAC 地址。
3. 避免重复展示同一个设备。
4. 支持停止扫描。
5. 扫描超时后自动停止。

### 4.3 连接设备

用户点击扫描列表中的设备后，App 发起 BLE 连接。

连接成功后需要：

1. 发现 GATT 服务。
2. 找到音立方主服务。
3. 找到音频 Notify、指令 Notify、Write 三个特征值。
4. 请求较大的 MTU，目标为 244 字节。
5. 订阅音频 Notify。
6. 订阅指令 Notify。
7. 更新界面连接状态。

### 4.4 指令发送

App 需要能通过 Write 特征值发送协议指令。

Demo 阶段至少实现以下指令：

| 功能 | App 指令 | 设备响应 |
| --- | --- | --- |
| 获取录音状态 | `YWT_BLE&STE` | `YWT_DEV&STE&VAL` |
| 开始录音 | `YWT_BLE&STA` | `YWT_DEV&STA&FNAME` |
| 停止录音 | `YWT_BLE&STO` | `YWT_DEV&STO` |
| 查询电量 | `YWT_BLE&BAT` | `YWT_DEV&BAT&RATE` |
| 查询存储 | `YWT_BLE&SPACE` | `YWT_DEV&SPA&F&T` |
| 查询固件版本 | `YWT_BLE&FW` | `YWT_DEV&FW&版本号` |

可选实现：

| 功能 | App 指令 |
| --- | --- |
| 获取设备 MAC | `YWT_BLE&MAC` |
| 获取设备时间 | `YWT_BLE&GT` |
| 设置设备时间 | `YWT_BLE&T&yyyyMMddHHmmss` |

### 4.5 指令响应解析

App 需要解析设备通过指令 Notify 返回的文本响应。

需要展示或保存的状态包括：

1. 当前是否正在录音。
2. 当前录音文件名。
3. 当前录音时长。
4. 电池电量。
5. 剩余存储容量。
6. 总存储容量。
7. 固件版本。
8. 错误状态，例如录音错误、存储已满、配对失败等。

### 4.6 实时音频数据接收

这是 Demo 的核心功能。

设备在录音状态下，会通过音频 Notify 通道实时推送 MP3 音频流。

App 需要做到：

1. 独立接收音频 Notify 数据。
2. 不把音频数据当作 UTF-8 文本解析。
3. 统计实时接收到的音频字节数。
4. 在界面展示音频数据接收情况。
5. 可以展示每秒接收速率、累计接收字节数、最近一次数据包大小。
6. 可选：将接收到的 MP3 数据保存成本地文件。
7. 可选：尝试播放实时音频流。

Demo 第一阶段可以先展示“已收到音频数据”和数据大小，不强制要求实时播放。

### 4.7 录音控制

App 界面需要提供录音控制按钮。

功能包括：

1. 点击“开始录音”发送 `YWT_BLE&STA`。
2. 收到 `YWT_DEV&STA&FNAME` 后，界面显示录音中和文件名。
3. 点击“停止录音”发送 `YWT_BLE&STO`。
4. 收到 `YWT_DEV&STO` 后，界面显示未录音。
5. 如果连接时设备已经在录音，收到 `YWT_DEV&RT&NAME&TIME` 后需要同步显示当前录音状态。

### 4.8 界面展示

Demo 页面需要清楚展示以下信息：

1. 蓝牙连接状态：未连接、扫描中、连接中、已连接、连接失败。
2. 扫描到的设备列表。
3. 当前连接的设备名称和 MAC 地址。
4. 电池电量。
5. 存储容量。
6. 固件版本。
7. 录音状态。
8. 当前录音文件名和录音时长。
9. 实时音频接收状态。
10. 通信日志。

界面不追求复杂设计，但要能清楚证明链路跑通。

## 五、非核心功能

以下功能协议中有定义，但 Demo 阶段可以先不做，除非后续时间充足：

1. 文件夹列表读取。
2. 录音文件列表读取。
3. BLE 同步历史录音文件。
4. WiFi 同步历史录音文件。
5. 删除设备录音文件。
6. WiFi 名称和密码管理。
7. Type-C 文件读取控制。
8. OTA 升级。
9. 设备重置和格式化。

这些功能属于完整产品能力，不是当前面试 Demo 的主线。

## 六、技术实现建议

建议项目分为三层：

| 层级 | 职责 |
| --- | --- |
| BLE 层 | 扫描、连接、服务发现、MTU 协商、Notify 订阅、Write 写入 |
| 协议层 | 保存 UUID、指令常量、响应常量 |
| 业务层 | 发送指令、解析响应、维护设备状态、分发音频数据 |
| UI 层 | 展示状态、设备列表、录音控制、日志和音频接收情况 |

关键注意事项：

1. 音频 Notify 和指令 Notify 必须分开。
2. 指令 Notify 返回的是文本，可以按 `YWT_DEV&...` 解析。
3. 音频 Notify 返回的是 MP3 二进制数据，需要按字节流处理。
4. BLE 单包大小受 MTU 限制，连接后应请求 244 字节 MTU。
5. 录音实时流优先保证接收和保存，播放可以作为增强功能。

## 七、验收标准

Demo 完成后，需要能现场演示以下流程：

1. 打开 App，授予蓝牙相关权限。
2. 点击扫描，看到音立方设备出现在列表中。
3. 点击设备，成功连接。
4. 连接成功后，界面显示已连接状态。
5. App 成功订阅指令 Notify 和音频 Notify。
6. 点击开始录音，设备开始录音，App 显示录音状态。
7. 设备录音时，App 能持续收到音频数据。
8. 页面能实时展示接收到的数据包大小、累计字节数或通信日志。
9. 点击停止录音，设备停止录音，App 状态同步更新。
10. 查询电量和存储时，App 能显示设备返回的数据。

最核心验收点：

**手机能连接外设，并实时接收到录音设备通过 BLE 推送的音频数据。**

## 八、后续增强方向

如果 Demo 基础功能完成，可以继续增强：

1. 保存实时 MP3 音频流到本地文件。
2. 增加实时播放能力。
3. 增加音频接收速率展示。
4. 增加断线重连。
5. 增加前台 Service，保证后台录音接收稳定。
6. 增加历史录音文件列表和下载功能。
7. 增加 WiFi 高速同步功能。

## 九、当前优先级

P0 必须完成：

1. BLE 扫描。
2. BLE 连接。
3. 订阅指令 Notify。
4. 订阅音频 Notify。
5. 发送开始录音、停止录音、查询电量、查询存储指令。
6. 接收并展示实时音频数据。

P1 尽量完成：

1. 保存接收到的 MP3 数据。
2. 展示实时接收速率。
3. 展示录音文件名和录音时长。
4. 错误提示。

P2 可选完成：

1. 实时播放音频。
2. 历史文件列表。
3. 文件下载。
4. WiFi 同步。
5. 自动重连。
